# X12 270/271 Eligibility Format

## Trigger Phrases

- 270
- 271
- eligibility inquiry
- eligibility check
- eligibility request
- eligibility response
- coverage verification
- benefit inquiry
- X12 eligibility

## Overview

X12 270 (Eligibility Inquiry) and 271 (Eligibility Response) are HIPAA-mandated standards for real-time verification of patient eligibility and benefits. This skill transforms HealthSim eligibility requests/responses into compliant 270/271 transactions.

## Transaction Types

| Transaction | Direction | Use Case |
|-------------|-----------|----------|
| 270 | Request | Provider asks payer for member eligibility |
| 271 | Response | Payer returns eligibility and benefit information |

## 270 Envelope Structure

```
ISA - Interchange Control Header
  GS - Functional Group Header
    ST - Transaction Set Header (270)
      BHT - Beginning of Hierarchical Transaction
      Loop 2000A - Information Source Level (Payer)
        HL - Hierarchical Level
        Loop 2100A - Information Source Name
          NM1 - Payer Name
      Loop 2000B - Information Receiver Level (Provider)
        HL - Hierarchical Level
        Loop 2100B - Information Receiver Name
          NM1 - Provider Name
          REF - Provider References
          N3 - Address
          N4 - Geographic Location
      Loop 2000C - Subscriber Level
        HL - Hierarchical Level
        TRN - Trace Number
        Loop 2100C - Subscriber Name
          NM1 - Subscriber Name
          REF - Subscriber References
          DMG - Subscriber Demographics
          DTP - Date Reference
        Loop 2110C - Subscriber Eligibility or Benefit Information
          EQ - Eligibility or Benefit Inquiry
          III - Relationship Information
          DTP - Date Reference
      Loop 2000D - Dependent Level (optional)
        HL - Hierarchical Level
        TRN - Trace Number
        Loop 2100D - Dependent Name
          NM1 - Dependent Name
          REF - Dependent References
          DMG - Demographics
        Loop 2110D - Dependent Eligibility Information
          EQ - Eligibility or Benefit Inquiry
    SE - Transaction Set Trailer
  GE - Functional Group Trailer
IEA - Interchange Control Trailer
```

## 271 Envelope Structure

```
ISA - Interchange Control Header
  GS - Functional Group Header
    ST - Transaction Set Header (271)
      BHT - Beginning of Hierarchical Transaction
      Loop 2000A - Information Source Level (Payer)
        HL - Hierarchical Level
        Loop 2100A - Information Source Name
          NM1 - Payer Name
          PER - Payer Contact
      Loop 2000B - Information Receiver Level (Provider)
        HL - Hierarchical Level
        Loop 2100B - Information Receiver Name
          NM1 - Provider Name
      Loop 2000C - Subscriber Level
        HL - Hierarchical Level
        TRN - Trace Number
        Loop 2100C - Subscriber Name
          NM1 - Subscriber Name
          REF - Subscriber References
          N3 - Address
          N4 - Geographic Location
          DMG - Demographics
          INS - Subscriber Relationship
        Loop 2110C - Subscriber Eligibility or Benefit Information
          EB - Eligibility or Benefit
          HSD - Health Care Services Delivery
          REF - Benefit References
          DTP - Benefit Dates
          AAA - Error Information
          MSG - Message Text
          Loop 2115C - Benefit-Related Entity Name
          Loop 2120C - Benefit-Related Provider
      Loop 2000D - Dependent Level (optional)
        [Similar structure to 2000C]
    SE - Transaction Set Trailer
  GE - Functional Group Trailer
IEA - Interchange Control Trailer
```

## Key Segments - 270 (Inquiry)

### ISA - Interchange Control Header
```
ISA*00*          *00*          *ZZ*PROVIDER_ID    *ZZ*PAYER_ID       *250115*1000*^*00501*000000001*0*P*:~
```

### GS - Functional Group Header
```
GS*HS*PROVIDER_ID*PAYER_ID*20250115*1000*1*X*005010X279A1~
```

| Element | Description | Value |
|---------|-------------|-------|
| GS01 | Functional ID | HS (Eligibility Inquiry) |
| GS08 | Version | 005010X279A1 |

### ST - Transaction Set Header
```
ST*270*0001*005010X279A1~
```

### BHT - Beginning of Hierarchical Transaction
```
BHT*0022*13*TRN20250115001*20250115*1000~
```

| Element | Position | Description | Values |
|---------|----------|-------------|--------|
| Structure Code | BHT01 | Hierarchical structure | 0022 |
| Purpose Code | BHT02 | Transaction purpose | 13=Request |
| Reference ID | BHT03 | Originator reference | Trace number |
| Date | BHT04 | Transaction date | CCYYMMDD |
| Time | BHT05 | Transaction time | HHMM |

### HL - Hierarchical Level
```
HL*1**20*1~      (Information Source - Payer)
HL*2*1*21*1~     (Information Receiver - Provider)
HL*3*2*22*0~     (Subscriber)
HL*4*3*23*0~     (Dependent - optional)
```

| Element | Description | Values |
|---------|-------------|--------|
| HL01 | Hierarchical ID | Sequential number |
| HL02 | Parent ID | Parent HL01 value |
| HL03 | Level Code | 20=Source, 21=Receiver, 22=Subscriber, 23=Dependent |
| HL04 | Child Code | 0=No children, 1=Has children |

### TRN - Trace Number
```
TRN*1*TRN20250115001*9012345678~
```

| Element | Description |
|---------|-------------|
| TRN01 | Trace Type (1=Current, 2=Referenced) |
| TRN02 | Trace Number |
| TRN03 | Originator ID |

### NM1 - Entity Name (Various Loops)
```
NM1*PR*2*BLUE CROSS BLUE SHIELD*****PI*BCBS001~        (Payer)
NM1*1P*2*SPRINGFIELD GENERAL HOSPITAL*****XX*1234567890~  (Provider)
NM1*IL*1*JOHNSON*MICHAEL*R***MI*MEM001234567~          (Subscriber)
```

| Entity ID | Description |
|-----------|-------------|
| PR | Payer |
| 1P | Provider |
| FA | Facility |
| IL | Insured (Subscriber) |
| 03 | Dependent |

## Validation Rules

### 270 Inquiry
1. **Required Segments**: ISA, GS, ST, BHT, HL, NM1 for payer/provider/subscriber
2. **TRN Required**: Trace number for tracking
3. **Valid NPI**: Provider NPI must be 10 digits
4. **Demographics**: Either DOB or Member ID required
5. **Service Type**: At least one EQ segment required

### 271 Response
1. **TRN Echo**: Must return original trace number
2. **EB Required**: At least one eligibility/benefit segment
3. **Coverage Dates**: Must include plan dates
4. **Error Handling**: AAA segment for any errors/rejections
5. **Amount Format**: Dollar amounts without decimal (500 = $5.00)
