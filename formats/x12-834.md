# X12 834 Benefit Enrollment Format

## Trigger Phrases

- 834
- X12 834
- enrollment file
- benefit enrollment
- member enrollment
- EDI enrollment
- enrollment transaction
- 834 file

## Overview

X12 834 is the HIPAA-mandated standard for electronic transmission of benefit enrollment and maintenance information. This skill transforms HealthSim enrollment entities into compliant 834 transactions.

## Transaction Types

| Code | Description | Use Case |
|------|-------------|----------|
| 00 | Original | Initial enrollment submission |
| 15 | Re-Enrollment | Annual enrollment updates |
| 21 | Addition | New member or dependent |
| 22 | Change | Modification to existing enrollment |
| 24 | Cancellation or Termination | End of coverage |
| 25 | Reinstatement | Restore cancelled coverage |

## Envelope Structure

```
ISA - Interchange Control Header
  GS - Functional Group Header
    ST - Transaction Set Header (834)
      BGN - Beginning Segment
      REF - Transaction Set Policy Number (optional)
      DTP - File Effective Date (optional)
      Loop 1000A - Sponsor Name (Employer/Group)
        N1 - Sponsor Name
        N2 - Additional Name (optional)
        N3 - Address
        N4 - Geographic Location
      Loop 1000B - Payer Name (Insurance Company)
        N1 - Payer Name
        N3 - Address
        N4 - Geographic Location
      Loop 1000C - TPA/Broker Name (optional)
        N1 - TPA Name
      Loop 2000 - Member Level Detail (repeating)
        INS - Insured Benefit
        REF - Member ID References
        DTP - Coverage Dates
        Loop 2100A - Member Name
          NM1 - Member Name
          PER - Contact Information
          N3 - Address
          N4 - Geographic Location
          DMG - Demographics
        Loop 2100B - Incorrect Member Name (for corrections)
        Loop 2100C - Member Mailing Address
        Loop 2100D - Employer Name
        Loop 2100E - School Name
        Loop 2100F - Custodial Parent
        Loop 2100G - Responsible Person
        Loop 2100H - Drop Off Location
        Loop 2200 - Disability Information (optional)
        Loop 2300 - Health Coverage (repeating)
          HD - Health Coverage
          DTP - Coverage Dates
          REF - Plan References
          Loop 2310 - Provider Information
          Loop 2320 - COB Information
    SE - Transaction Set Trailer
  GE - Functional Group Trailer
IEA - Interchange Control Trailer
```

## Key Segments

### ISA - Interchange Control Header
```
ISA*00*          *00*          *ZZ*SENDER_ID      *ZZ*PAYER_ID       *250115*1430*^*00501*000000001*0*P*:~
```

### GS - Functional Group Header
```
GS*BE*SENDER_ID*PAYER_ID*20250115*1430*1*X*005010X220A1~
```

| Element | Description | Value |
|---------|-------------|-------|
| GS01 | Functional ID | BE (Benefit Enrollment) |
| GS08 | Version | 005010X220A1 |

### ST - Transaction Set Header
```
ST*834*0001*005010X220A1~
```

### BGN - Beginning Segment
```
BGN*00*ENR20250115001*20250115*1430****2~
```

| Element | Position | Description | Values |
|---------|----------|-------------|--------|
| Transaction Set Purpose | BGN01 | Purpose code | 00=Original, 15=Re-enrollment, etc. |
| Reference ID | BGN02 | Transaction reference | Unique identifier |
| Date | BGN03 | Transaction date | CCYYMMDD |
| Time | BGN04 | Transaction time | HHMM |
| Action Code | BGN08 | Action | 2=Change, 4=Verify, RX=Replace |

### N1 - Sponsor (Loop 1000A)
```
N1*P5*ACME CORPORATION*FI*123456789~
```

| Element | Description | Values |
|---------|-------------|--------|
| N101 | Entity ID | P5=Plan Sponsor |
| N102 | Name | Organization name |
| N103 | ID Qualifier | FI=Federal Tax ID, 24=EIN |
| N104 | Identification Code | Tax ID/EIN |

### N1 - Payer (Loop 1000B)
```
N1*IN*BLUE CROSS BLUE SHIELD*FI*987654321~
```

| Element | Description | Values |
|---------|-------------|--------|
| N101 | Entity ID | IN=Insurer |
| N102 | Name | Payer name |
| N103 | ID Qualifier | FI=Federal Tax ID |
| N104 | Identification Code | Payer tax ID |

### INS - Insured Benefit (Loop 2000)
```
INS*Y*18*021*28*A***FT~
```

| Element | Position | Description | Values |
|---------|----------|-------------|--------|
| Yes/No Response | INS01 | Member indicator | Y=Yes, N=No (termination) |
| Relationship Code | INS02 | Relationship | 18=Self, 01=Spouse, 19=Child |
| Maintenance Type | INS03 | Action type | 021=Addition, 024=Cancel, 025=Reinstatement |
| Maintenance Reason | INS04 | Reason code | See below |
| Benefit Status | INS05 | Coverage status | A=Active, C=COBRA, S=Surviving Spouse |
| Employment Status | INS08 | Employment | FT=Full-time, PT=Part-time, RT=Retired |

#### Maintenance Reason Codes (INS04)
| Code | Description |
|------|-------------|
| 01 | Divorce |
| 02 | Birth |
| 03 | Death |
| 04 | Retirement |
| 05 | Adoption |
| 07 | Termination of Benefits |
| 08 | Termination of Employment |
| 11 | Disability |
| 14 | Voluntary Withdrawal |
| 20 | Initial Enrollment |
| 21 | Re-enrollment |
| 22 | Leave of Absence with Benefits |
| 25 | Change in Identifying Data Elements |
| 28 | Initial Enrollment - New Hire |
| 32 | Open Enrollment |
| 33 | Dependent Aged Out |
| 41 | Re-Enrollment |
| XN | Notification Only |

### REF - Member Reference
```
REF*0F*MEM001234567~
REF*1L*GRP001234~
REF*23*XYZ123456789~
```

| Qualifier | Description |
|-----------|-------------|
| 0F | Subscriber Number |
| 1L | Group Number |
| 23 | Client Number |
| DX | Department Number |
| ZZ | Mutually Defined |

### DTP - Date/Time Reference
```
DTP*336*D8*20250201~
DTP*337*D8*20251231~
DTP*300*D8*20250115~
```

| Qualifier | Description |
|-----------|-------------|
| 300 | Enrollment Signature Date |
| 303 | Maintenance Effective Date |
| 336 | Employment Begin |
| 337 | Employment End |
| 338 | Medicare Begin |
| 339 | Medicare End |
| 340 | COBRA Begin |
| 341 | COBRA End |
| 350 | Coverage Begin |
| 351 | Coverage End |
| 356 | Eligibility Begin |
| 357 | Eligibility End |

### NM1 - Member Name (Loop 2100A)
```
NM1*IL*1*JOHNSON*MICHAEL*R***34*123456789~
```

| Element | Position | Description | Values |
|---------|----------|-------------|--------|
| Entity ID | NM101 | Entity type | IL=Insured |
| Entity Type | NM102 | Person/Org | 1=Person, 2=Organization |
| Last Name | NM103 | Family name | |
| First Name | NM104 | Given name | |
| Middle Name | NM105 | Middle name/initial | |
| ID Qualifier | NM108 | ID type | 34=SSN, MI=Member ID |
| ID | NM109 | ID value | SSN or Member ID |

### PER - Contact Information
```
PER*IP**TE*5552345678*EM*michael.johnson@email.com~
```

| Element | Description |
|---------|-------------|
| PER01 | Contact function (IP=Insured Party) |
| PER03/04 | Communication type and number |
| PER05/06 | Additional contact method |

### N3/N4 - Address
```
N3*456 OAK AVENUE~
N4*SPRINGFIELD*IL*62702*USA~
```

### DMG - Demographics
```
DMG*D8*19850315*M**:RET:2106-3~
```

| Element | Position | Description | Values |
|---------|----------|-------------|--------|
| Date Format | DMG01 | Date qualifier | D8=CCYYMMDD |
| Birth Date | DMG02 | Date of birth | CCYYMMDD |
| Gender | DMG03 | Gender code | M, F, U |
| Race/Ethnicity | DMG05 | Composite | Code set values |

### HD - Health Coverage (Loop 2300)
```
HD*021**HLT*PPO-GOLD*FAM~
```

| Element | Position | Description | Values |
|---------|----------|-------------|--------|
| Maintenance Type | HD01 | Action | 021=Add, 024=Cancel, 030=Change |
| Insurance Line | HD03 | Coverage type | HLT=Health, DEN=Dental, VIS=Vision |
| Plan Code | HD04 | Plan identifier | Plan code |
| Coverage Level | HD05 | Tier | EMP, ESP, ECH, FAM |

#### Insurance Line Codes (HD03)
| Code | Description |
|------|-------------|
| HLT | Health |
| DEN | Dental |
| VIS | Vision |
| HEA | Hearing |
| LIF | Life |
| STD | Short-Term Disability |
| LTD | Long-Term Disability |
| FSA | Flexible Spending Account |
| HSA | Health Savings Account |
| HRA | Health Reimbursement Account |

#### Coverage Level Codes (HD05)
| Code | Description |
|------|-------------|
| EMP | Employee Only |
| ESP | Employee + Spouse |
| ECH | Employee + Children |
| FAM | Employee + Family |
| TWO | Employee + One Dependent |

## Validation Rules

1. **ISA/IEA Control Numbers**: Must match and be unique
2. **GS/GE Control Numbers**: Must match within interchange
3. **ST/SE Control Numbers**: Must match within functional group
4. **Segment Count**: SE01 must equal actual segment count
5. **Required Segments**: BGN, N1 (sponsor/payer), INS, NM1 for each member
6. **Date Formats**: All dates must be CCYYMMDD format
7. **SSN Format**: Must be 9 digits without dashes
8. **Relationship Consistency**: INS02 must match member's actual relationship
9. **Coverage Dates**: Start date must be before end date
10. **Member Count**: Each INS segment must have corresponding NM1
